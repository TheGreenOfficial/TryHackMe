**The OSI Model:** <br><br>
**The OSI model breaks networking into 7 layers, from physical hardware up to applications, each handling a specific role like data transmission, routing, formatting, and communication management.** 
<br><br>
**Questions:** <br>
**1) Which layer would choose to send data over TCP or UDP? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 4**
<br><br>
**2) Which layer checks received information to make sure that it hasn't been corrupted? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 2**
<br><br>
**3) In which layer would data be formatted in preparation for transmission? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 2**
<br><br>
**4) Which layer transmits and receives data? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 1**
<br><br>
**5) Which layer encrypts, compresses, or otherwise transforms the initial data to give it a standardised format? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 6**
<br><br>
**6) Which layer tracks communications between the host and receiving computers? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 5**
<br><br>
**7) Which layer accepts communication requests from applications? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 7**
<br><br>
**8) Which layer handles logical addressing? Answer with the number of the layer: e.g. if the answer would be "the application layer", then you would enter "7".** <br>
**-> 3**

**9) When sending data over TCP, what would you call the "bite-sized" pieces of data?** <br>
**-> Segments**

**-> 7**

**9) Which transport layer protocol would be best suited to transmit a live video?** <br>
**-> UDP**

**Encapsulation** <br><br>
**Encapsulation wraps data with layer-specific headers (and a trailer) as it moves down the OSI layers for transmission, and de-encapsulation reverses this process on receipt to reconstruct the original data.**
<br><br>
**Questions:** <br>
**1) How would you refer to data at layer 2 of the encapsulation process (with the OSI model)?** <br>
**-> Frames**
<br><br>
**2) How would you refer to data at layer 4 of the encapsulation process (with the OSI model), if the UDP protocol has been selected?** <br>
**-> Datagrams**
<br><br>
**3) What process would a computer perform on a received message?** <br>
**-> De-encapsulation**
<br><br>
**4) Which is the only layer of the OSI model to add a trailer during encapsulation?**
**-> Data Link**
<br><br>
**5) Does encapsulation provide an extra layer of security (Aye/Nay)?**
**-> Aye**

**The Tcp/Ip model** <br><br>
**The TCP/IP model simplifies networking into four layers and forms the foundation of real-world communication, using protocols like TCP and IP to manage data transfer, addressing, and reliable connections through mechanisms like the three-way handshake.**
<br><br>
**Questions:** <br>
**1) Which model was introduced first, OSI or TCP/IP?** <br>
**-> TCP/IP**
<br><br>
**2) Which layer of the TCP/IP model covers the functionality of the Transport layer of the OSI model (Full Name)?** <br>
**-> Transport**
<br><br>
**3) Which layer of the TCP/IP model covers the functionality of the Session layer of the OSI model (Full Name)?** <br>
**-> Application**
<br><br>
**4) The Network Interface layer of the TCP/IP model covers the functionality of two layers in the OSI model. These layers are Data Link, and?.. (Full Name)?** <br>
**-> Physical**
<br><br>
**5) Which layer of the TCP/IP model handles the functionality of the OSI network layer?** <br>
**-> Internet**
<br><br>
**6) What kind of protocol is TCP?** <br>
**-> Connection-based**
<br><br>
**7) What is SYN short for?** <br>
**-> Synchronise**
<br><br>
**8) What is the second step of the three way handshake?** <br>
**-> SYN/ACK**
<br><br>
**9) What is the short name for the "Acknowledgement" segment in the three-way handshake?** <br>
**-> ACK**

**Ping** <br><br>
**The ping command is a simple networking tool that uses ICMP to check connectivity to a target and can also reveal its IP address.**
<br><br>
**Questions:** <br>
**1) What command would you use to ping the bbc.co.uk website?** <br>
**-> ping bbc.co.uk**
<br><br>
**2) Ping muirlandoracle.co.uk. What is the IPv4 address?** <br>
**-> 217.160.0.152**
<br><br>
**3) What switch lets you change the interval of sent ping requests?** <br>
**-> -i**
<br><br>
**4) What switch would allow you to restrict requests to IPv4?** <br>
**-> -4**
<br><br>
**5) What switch would give you a more verbose output?** <br>
**-> -v**

**Dig** <br><br>
**DNS (Domain Name System) translates domain names into IP addresses through a hierarchical lookup process involving local cache, recursive servers, root servers, TLD servers, and authoritative servers, with tools like dig allowing manual queries and TTL controlling cache validity.**
<br><br>
**Questions:** <br>
**1) What is DNS short for?** <br>
**-> Domain Name System**
<br><br>
**2) What is the first type of DNS server your computer would query when you search for a domain?** <br>
**-> Recursive**
<br><br>
**3) What type of DNS server contains records specific to domain extensions (i.e. .com, .co.uk, etc)*? Use the long version of the name.** <br>
**-> Top-Level Domain**
<br><br>
**4) Where is the very first place your computer would look to find the IP address of a domain?** <br>
**-> Hosts File**
<br><br>
**5) [Research] Google runs two public DNS servers. One of them can be queried with the IP 8.8.8.8, what is the IP address of the other one?** <br>
**-> 8.8.4.4**
<br><br>
**6) If a DNS query has a TTL of 24 hours, what number would the dig query show?** <br>
**-> 86400**
